# Chapter 8 – Management and Governance

## Overview
This chapter introduces the AWS services used to monitor, manage, and govern your cloud environment. You'll learn how to improve operational visibility, automate administration tasks, and maintain security and compliance across your AWS resources.

## In this chapter
- Understand the purpose of AWS management and governance services.
- Monitor applications and infrastructure using Amazon CloudWatch.
- Audit AWS activity with AWS CloudTrail.
- Review best practice recommendations using AWS Trusted Advisor.
- Manage EC2 instances using AWS Systems Manager.
- Explore operational best practices for managing AWS environments.

## Learning Outcomes
By the end of this chapter, you will be able to:
- Monitor AWS resources using Amazon CloudWatch.
- Track API activity and auditing with AWS CloudTrail.
- Use AWS Trusted Advisor to identify security, performance, and cost optimization opportunities.
- Remotely manage EC2 instances using AWS Systems Manager.
- Apply AWS operational best practices to improve governance.

## Labs
- Create and customize an Amazon CloudWatch dashboard.
- Execute commands on EC2 instances using AWS Systems Manager Run Command.
- Configure and use AWS Systems Manager Session Manager.
- Explore AWS CloudTrail by tracking and reviewing AWS API activity.
